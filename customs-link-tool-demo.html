📋 产品经理标注（PRD交互说明 + 数据逻辑）

一、检索工作台 - 交互逻辑

  • 交互SKU输入框：单击输入单个SKU；双击弹出批量输入弹窗（支持分号/逗号/空格/回车分隔，上限100个）
  • 交互目的国：单选下拉，决定检索哪个站点的竞品（DE/FR/IT/ES/UK/US）。多SKU共享同一目的国
  • 交互搜索按钮点击后：显示2步进度条（Step1创建任务 → Step2等待返回），实际耗时约5分钟
  • 数据调用 Sorftime SimilarProductRealtimeRequest 接口，传入SKU主图base64 + 目的国domain参数
  • 数据接口为异步模式：① 创建任务 → ② 轮询状态 → ③ 获取结果（SimilarProductRealtimeRequestCollection）
  • 数据返回结果过滤：剔除无价格的ASIN，每个SKU按价格升序取前3条展示

二、检索结果 - 展示逻辑

  • 交互结果按SKU分组展示，组间用粗分隔线区分。每组内按价格从低到高排列
  • 交互最低价行：背景高亮（浅绿）+ 价格后标注"★最低"
  • 交互SKU主图和竞品主图均支持点击放大预览 + 下载
  • 交互销售链接完整展示，支持一键复制（📋）和跳转亚马逊（↗）
  • 交互不分页，检索结果一次性全量展示

三、推送清关 - 权限与逻辑

  • 权限"推送清关"按钮：关务专员 + 产品开发可操作，前端按ERP系统角色权限配置控制显示（已支持菜单/按钮/字段级权限）
  • 数据操作人：取ERP系统当前登录用户昵称
  • 交互单行推送：点击"推送清关"→ 该行最后一列实时写入推送日志（时间+操作人+原链接+新链接）
  • 交互"一键推送最低价链接"：点击后弹出确认弹窗，展示待推送的SKU/ASIN/价格列表 + 警告"直接覆盖清关基础信息，不可撤销"。确认后批量执行
  • 数据推送逻辑：调用物流系统-清关基础信息-详情属性资料-清关审核模块，覆盖对应SKU的销售链接字段
  • 数据销售链接展示规则：按时间降序展示最新3条推送记录
  • 数据推送日志格式："产品库-销售链接推送覆盖 / SKU / 目的国 / 原销售链接 / 新销售链接+价格"
  • 数据日志持久化：写入操作日志表，关联SKU+操作人+时间，检索日志中可查

四、检索日志 - 数据逻辑

  • 交互支持按 SKU/ASIN、操作人、时间范围 筛选查询
  • 交互点击"查看详情"弹窗展示该次检索的完整返回数据（含图片对比+链接+推送日志列）
  • 数据检索记录不设数据权限，所有人可见所有历史检索结果
  • 数据每条记录关联：检索时间、操作人、SKU列表、目的国、匹配成功/失败数、推送情况

五、边界场景处理

  • 数据SKU无主图：直接过滤掉，不计入前端显示个数（无主图无法以图搜图，后端静默跳过）
  • 交互匹配结果为0：该SKU行展示"未匹配到竞品"灰色提示文字，不隐藏SKU
  • 数据匹配结果无价格：直接过滤掉，不计入前端显示个数（与现有逻辑一致）
  • 交互SKU不存在（ERP中查无此编码）：点击"搜索竞品"按钮时触发提示"以下SKU在系统中不存在：xxx"
  • 数据重复SKU：查询自动去重，相同SKU仅执行一次检索
  • 数据接口超时/失败：请后端评估重试机制和超时阈值，前端展示"检索失败，请重试"提示

六、导出Excel

  • 交互导出按钮与"一键推送最低价链接"并列，所有角色均可操作
  • 数据导出范围：当前前端所展示的检索结果数据（非接口全量）
  • 数据导出字段：SKU编码、SKU主图URL、匹配商品名、ASIN、销售链接、价格（当地币）、目的国、检索时间
  • 数据文件名规则：竞品检索结果_{目的国}_{yyyyMMdd_HHmm}.xlsx
  • 交互最低价行在Excel中标记★（价格字段前加★符号）
  • 权限无权限限制，所有角色均可导出

七、2期规划

  • 2期导入检索：支持Excel模板导入（字段：SKU编码、国家），导入后自动执行检索流程，交互与手动输入一致
  • 2期工具扩展：竞品检索可复用为选品系统组件，支持更多维度的竞品筛选（价格区间、评分、销量等）
