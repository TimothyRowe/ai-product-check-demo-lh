# AI 检测功能修改说明

## 背景
开品资料维度是 SPU，下面可能有多个变体 SKU。检测结果需要精确到 SKU 级别。

## 数据结构划分

### SPU 级别信息
- SPU 商品名称（中文/英文）
- 商品目录
- 平台类目
- 是否季节产品
- 是否节庆产品
- 主竞品链接
- 开品备注
- 开发人员
- SPU 商品图片

### SKU 级别信息（每个 SKU 独立）
1. **SKU 信息**
   - 属性标签、SKU 商品名称、是否定制、开发类型
   - 关键词、商品参数、商品描述、卖点描述
   - 包装清单、注意事项、是否有适配车型
   - 产品线名称、是否工厂限价、是否刷单 SKU

2. **物流资料**
   - 报关名称、材质、用途、目的仓订单属性
   - 销售国家、是否能做商检
   - 基础属性、插头属性、电压属性、电池属性
   - 形态属性、违禁属性、特殊属性、附加属性
   - 头程运输方式、插头型号

3. **合规信息**
   - 品牌类型、品牌名称、是否有型号
   - 是否有 LOGO、是否侵权、是否违禁

4. **包装信息**
   - 单个商品包装尺寸/毛重/净重/尺寸
   - 外箱尺寸/毛重、装箱规格、发货属性

5. **供应商信息**
   - 采购单价、供应状态、订单交期
   - 最小起订量、供应商名称

## 需要修改的文件

### 1. ai-config-demo.html（校验规则配置）
**修改点：**
- 在"校验规则"部分，每条规则需要标注适用维度（SPU / SKU）
- 示例：
  - 维度1（电气属性 × 销售国家）→ **SKU 级别**（因为涉及插头、电压、销售国家）
  - 维度2（包装尺寸 × 重量逻辑）→ **SKU 级别**（包装信息属于 SKU）
  - 维度3（电池 × 运输合规）→ **SKU 级别**（电池属性属于物流资料）
  - 维度4（物流形态 × 运输方式）→ **SKU 级别**（物流资料）
  - 维度5（合规信息 × 侵权风险）→ **SKU 级别**（合规信息）
  - 维度6（产品描述 × 结构化字段）→ **SKU 级别**（商品描述属于 SKU 信息）
  - 维度7（供应商 × 采购逻辑）→ **SKU 级别**（供应商信息）
  - 如果有 SPU 级别的规则（如"SPU 名称长度"），需要单独标注

**输出 JSON 格式调整：**
```json
{
  "spu_issues": [
    {
      "dimension": "SPU名称规范",
      "severity": "warning",
      "title": "SPU产品名称过长",
      "description": "...",
      "related_fields": ["SPU商品名称(中文)"]
    }
  ],
  "sku_issues": [
    {
      "sku_code": "MG260009801",
      "sku_name": "含2电池68件套 美规",
      "country": "美国",
      "issues": [
        {
          "dimension": "电气属性×销售国家",
          "severity": "error",
          "title": "插头类型与销售国家不匹配",
          "description": "...",
          "related_fields": ["销售国家", "插头属性", "插头型号"]
        }
      ]
    }
  ]
}
```

### 2. ai-check-demo.html（检测结果弹窗）
**修改点：**
- 右侧检测报告面板，改为 SPU/SKU 分层展示
- 参考 ai-check-log-demo.html 的实现
- SPU 问题用蓝色标题栏
- SKU 问题用灰色标题栏，显示 SKU 编号 + 名称 + 销售国家
